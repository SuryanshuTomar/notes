Q11: How does variable hoisting interact with function declarations and expressions?

Function declarations are hoisted and can be called before their definition. Function expressions (assigned to var) are hoisted as variables (initialized as undefined), so calling them before assignment throws a TypeError.

console.log(hoistedFunc()); // 'I am hoisted'
// console.log(exprFunc()); // TypeError: exprFunc is not a function

function hoistedFunc() {
    return 'I am hoisted';
}

var exprFunc = function() {
    return 'I am an expression';
};
